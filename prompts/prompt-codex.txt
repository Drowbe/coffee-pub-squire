JSON TEMPLATE FOR CODEX

I want you to build a JSON array that will be used for a codex feature in my foundryvtt module using an "import json" feature. This JSON will be based on the criteria I will share below. Here are the rules for how you will add data to the JSON.

**NAME** - The name of the entry. DO not add "the" or "a" to the name.
**CATEGORY** - This will be the organizing and grouping mechanism for the entry, so be smart about it as we do not want a bunch of similar categories. They should be unique and specific. Any non-unique classifications would be better as tags. It should be plural as there will be multiple things in the category. e.g. "Characters". Prefer these established categories when they fit: "Characters", "Establishments", "Events", "Factions", "Items", "Artifacts", "Landmarks", "Locations", "Maps". Place-related entries MUST be split rather than dumped into Locations: use "Establishments" for businesses and services (shops, inns, taverns, guildhalls, temples-as-businesses), "Landmarks" for distinctive named sites and features (bridges, waterfalls, monuments, ruins, standing stones, districts with a proper name), and reserve "Locations" for broad geography only (regions, settlements, wilderness areas).
**SUMMARY** - A concise summary of the entry that would help someone understand what, where, or who this is, and enough context to make it interesting. (make it 200 to 500 characters)
**PLOTHOOK** - The relationship to the plot,  especially if they have something the party might need (under 200 characters)
**LOCATION** - the entry's place in the world as a hierarchical path with " > " separators, ordered broadest to most specific: Realm > Region > Site > Area. Realm is the world or continent (e.g. "Faerûn"), Region is the territory or geographic area (e.g. "Moonsea", "Daggerdale"), Site is the settlement or named place (e.g. "Teshwave", "Dagger Fall"), Area is the specific spot within it (e.g. "The Broken Anvil", "Aquatic Crypt"). Include only the levels you actually know — "Faerûn > Moonsea > Teshwave" is valid without an Area. Reuse the exact same spelling for shared levels across entries so they group together.
**TAGS** - a list of tags that would help filter this entry when looking it up. These will be used for filtering, so characteristics and identifying attributes like type, location, faction, etc., would be useful. There should never be tags for words like "the" and there should never be single-letter tags like "a". The first tag should always be the category. Add no more than 5 tags. Having spaces in the tag is okay, but do not divide words with special characters like underscores. You should always add the location as a tag, but the location should be specific. Something like "Phlan - Thorne Island - Aquatic Crypt" would be three tags. Also, be mindful of when a particular tag might need to be a second, less specific tag. For instance, "black cult of the dragon" is a particular tag, but we should add a second tag for "cult" which would be another proper tag. They should be formatted to be json-friendly and will be an array formatted like: "npc", "inn", "drinking game", "informant", "phlan". A tag would be something that would likely be applied to more than one entry. For example, "arena beast" is unnecessary... it should be "arena" and "beast". They should not be niche or overly specific phrases. Remember, these are used to group like things based on characteristics of the entry.
**EXPANDEDDETAILS** - OPTIONAL rich lore for the entry, rendered on the full journal page below the summary and field block. Use simple HTML: <h2> or <h3> section headings (for example Geography, Government, History for a location, or Appearance, Motivations, Relationships for a character), and <p> paragraphs. Do not add a heading named "Expanded Details" — the page already labels this section. Do not include <img> tags or scripts. Escape the HTML so it is JSON-friendly (single-line string, double quotes escaped). If there is not enough source material for deeper detail, set this to null rather than padding it.

**LINKS** - OPTIONAL cross-references from this entry to other documents that exist in the world or in the campaign's compendiums. Each link is an object with a "name" (the document's plain name, spelled exactly as it appears on the document) and a "type" (one of: "actor" for creatures and NPCs, "item" for objects, gear, and artifacts, "spell", "feature", "journal" for lore pages and locations, or "rolltable"). Do NOT invent UUIDs — write the plain name and the importer resolves it against the GM's configured compendiums. Only add a link when you are confident a document by that name actually exists; a name that does not resolve is dropped and reported to the GM. Use this for things the entry references — the faction an NPC belongs to, an artifact a location houses, the NPC who runs an establishment. You do not need to link the entry to itself: the importer already resolves the entry's own name using its category.
